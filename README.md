# 🤖 Speexx Bot 
ง่ายๆ คือ... เป็นบอตที่ช่วยทำแบบฝึกหัดเรียนภาษาบน Speexx ให้เสร็จเองเอง 😂

##ทำไมต้องใช้?
ถ้าเคยทำแบบฝึกหัดบน Speexx มาแล้ว ก็รู้ว่า... หนึ่งบทเรียนมีแต่แบบฝึกหัดเยอะแยะ ต้องกดบ่อยมากก ตัวบอตนี้ช่วยกดให้เสร็จเอง

## มี 2 เวอร์ชั่น

### 📱 Mobile Version - Speexx-Mobile.js
- เป็น userscript ใช้ได้กับ Kiwi Browser ที่มี Tampermonkey
- ติดตั้งแล้วเปิดเว็บ Speexx ขึ้นมา ตัวมันก็เริ่มทำอัตโนมัติ
- ไม่ต้องรัน command อะไร ด้วย

### 🖥️ Desktop Version - speexx-pc.js  
- เป็น Node.js script ใช้ Puppeteer เปิดเบราว์เซอร์จริงๆ
- เก็บ login session ได้ ลงครั้งแรก แล้วครั้งหน้ารันแล้วเข้าสู่ระบบอัตโนมัติ
- เหมือนจำลองคนจริงใช้เบราว์เซอร์

## วิธีการใช้งาน?
ทั้งสองเวอร์ชั่นมันทำเหมือนกัน:

1. **เข้า Speexx portal** → ล็อกอิน
2. **เปิดแบบฝึกหัด** → บอตเริ่มทำ
3. **ทำคำตอบ** → ทริก event ของ Backbone.js (ระบบใน Speexx)
4. **ส่งคำตอบ** → ไปขั้นตอนนิดนึง
5. **กดถัดไป** → ไปแบบฝึกหัดตัวถัดไป
6. **ทำซ้ำ** จนเสร็จทั้งหมด 🎉

## เรื่อง "Event Triggering" 

สิ่งที่ทำให้บอตนี้ดีกว่าการกด DOM  ปกติ คือ มันไม่ได้กด button ปกติ แต่ กดโดยตรงผ่าน Backbone.js event

ปกติหลายคนจะทำแบบนี้:
---javascript
document.querySelector('button').click(); // กด DOM

แต่บอตนี้ทำแบบนี้:
---javascript
window.entryp.trigger("correct");  // ทริก event โดยตรง
window.entryp.trigger("solve");    // ส่งเหมือนระบบคุยเองกับตัวเอง
// เพราะ Speexx ใช้ Backbone.js 

## Timeline ที่เกิดขึ้น
```
0ms    → ส่ง event "correct" (ยังไม่ส่งคำตอบ)
1500ms → ส่ง event "solve" (ส่งคำตอบ)
4000ms → ส่ง event "correct" (ยืนยันแล้ว)
6000ms → กดปุ่มถัดไป (ไปข้อถัดไป)
```

## ⚠️ หมายเหตุที่สำคัญ

- โปรเจ็กต์นี้เป็นเพื่อการศึกษาเท่านั้น เข้าใจการทำงานของ web automation กับ Backbone.js
- อย่าใช้ในทำเพื่อเสียหาย ❌
- Speexx ระบบอาจอัพเดตหลังจากนี้ บอตอาจไม่ทำงาน
- ไม่รับผิดชอบต่อความเสียหายที่เกิดขึ้นั้งสิ้น

## ที่ใช้
- **Node.js + Puppeteer** (Desktop version) - จำลองเบราว์เซอร์จริง
- **Tampermonkey** (Mobile version) - userscript
- **Backbone.js event system** - คุยกับระบบเกมโดยตรง
